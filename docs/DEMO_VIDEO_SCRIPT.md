# Demo video script (target: 2 minutes 40 seconds)

The final video must be public on YouTube, shorter than three minutes, and include audible narration. Record the app at 1280×720 or 1920×1080 with browser zoom at 100%.

## 0:00–0:15 — The problem

**Show:** Project thumbnail, then the home page.

**Narration:**

> Streaming platforms know what is popular, but popularity does not explain whether one specific person will enjoy a title. Cinema Taste turns a real viewing history into an explainable personal signal.

## 0:15–0:35 — The local profile

**Show:** Home statistics, then open **بصمة الذائقة**.

**Narration:**

> The application keeps liked, disliked, and watchlist records in portable local files. It builds a taste fingerprint from genres, directors, personal ratings, and high-rated reference works.

## 0:35–1:05 — OMDb handoff

**Show:** Open **إضافة عمل**, search for a title in OMDb, select it, and reveal the review form. Select **قياس مدى القابلية**.

**Narration:**

> OMDb supplies public metadata for the selected movie or series. After I review the details, one button sends that same metadata directly to the dedicated Likelihood page, so the user never has to search twice.

## 1:05–1:40 — Likelihood result

**Show:** The **مدى القابلية** result: percentage, confidence, reasons, liked comparisons, and caution references.

**Narration:**

> The local engine compares the title with my actual history. It returns a predicted percentage, a confidence level, supporting reasons, the closest works I liked, and caution signals from works I disliked. This is deliberately an informed starting point, not an opaque yes-or-no verdict.

## 1:40–1:58 — Judge-friendly demo

**Show:** Refresh the Likelihood page and select **تجربة فورية بلا مفتاح**.

**Narration:**

> Judges can test the core experience instantly without an OMDb key. The included demo performs the full analysis locally and makes no external request.

## 1:58–2:28 — Codex and GPT-5.6

**Show:** Expand **English judge summary**, then select **Copy English brief**. Briefly show the repository README and automated test output.

**Narration:**

> I used Codex with GPT-5.6 to isolate the contest version, design the explainable analysis engine, build the direct OMDb handoff, create automated engine and API tests, and verify the real interface at 1280 by 720. The application also generates a structured evidence brief for a deeper GPT-5.6 discussion without exposing an OpenAI API key.

## 2:28–2:40 — Close

**Show:** Return to the home page, then end on the thumbnail.

**Narration:**

> Cinema Taste makes recommendations personal, explainable, and locally controlled. It does not only predict what may fit your taste. It shows why.

## Recording checklist

- Keep the final duration below 3:00.
- Use the English narration above, or provide complete English captions if narrating in Arabic.
- Do not show the OMDb key, private browser tabs, notifications, or personal account details.
- Confirm that the YouTube visibility is **Public**, not Unlisted or Private.
- Put the Devpost project URL and repository URL in the YouTube description.
